# Learning

## Scope Defaults
Treat learn, study, understand, course, skill, and tutorial requests as move-from-current-knowledge-to-working-competence. Start by assessing current knowledge level and available time. Default to structured, project-based, active learning over passive reading. Prefer free and open resources unless user specifies budget. Exclude university enrollment decisions, professional certification advice, or long-term career strategy unless explicitly approved.

Default scope includes: current level assessment, clear goal definition, week-by-week milestone plan, recommended resources (free-first), one hands-on project to build during learning, and a measurable completion checkpoint.

## Checklist
- Assess current level: complete beginner, some exposure, or intermediate?
- State the goal: what should the user be able to do when finished?
- Break into milestones: 2-4 week chunks, each with a concrete deliverable.
- Recommend 1-2 resources per milestone (video course, book, docs, tutorial).
- Include one hands-on project that uses skills from each milestone.
- Add a progress check: how to know you've reached each milestone.
- Note realistic time estimate per week needed to hit the goal.

## Pitfalls
- Prescribing passive resources (just watch videos) without active practice.
- Underestimating the user's current level and wasting time on basics they know.
- Overloading with too many resources per topic — pick the best one.
- No measurable milestone — "understand Python" is not a milestone, "build a working to-do CLI" is.
- Ignoring time constraints — a 6-month plan for a user who has 2 hours per week.

## Success Looks Like
The user has a clear week-by-week plan, each week ending in something they can do or build. Resources are specific (link, title, chapter). The final milestone is a project they can show or use. Time estimate is honest. The plan is adjustable based on how fast they move.

## Examples
Example 1: "Learn Python." Assess level (assume beginner). Goal: build a small working project in 8 weeks at 5 hrs/week. Week 1-2: Python basics (Automate the Boring Stuff, free). Week 3-4: functions, files, error handling. Week 5-6: simple CLI project. Week 7-8: refine and add one feature. Milestone check: can you write a script that reads a file and outputs a summary?

Example 2: "Understand machine learning." Assess level (assume technical, no ML experience). Goal: understand supervised learning and run a real model. Week 1: what is ML + linear regression (fast.ai lesson 1). Week 2: train a model on tabular data. Week 3: understand what went wrong and fix it. Milestone: working Kaggle submission.
